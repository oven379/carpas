# Внутренняя SEO-настройка КарПас (черновик под ваши ответы)

Документ для команды: что сделано в коде, что решить на продакшене, ответы по пунктам анкеты и открытые вопросы.

## Уже в репозитории

- **`react-helmet-async`**: `<title>`, `meta description`, `robots`, `canonical`, Open Graph, Twitter Card, JSON-LD.
- **`CabinetRouteSeo`**: для маршрутов кабинета и входа — **noindex**, отдельные заголовки по пути.
- **Публичные страницы**: свой SEO — `/about`, `/d/:id`, `/g/:slug`, `/share/:token` (в т.ч. при загрузке — промежуточный title/description).
- **Главная `/about`**: разметка **Organization**, **WebSite**, **FAQPage** (тексты согласованы с блоком FAQ на странице).
- **`/d/:id`**: **BreadcrumbList** + **AutomotiveBusiness** (если есть абсолютный origin).
- **`public/robots.txt`**: закрыт кабинет, открыты публичные префиксы; строку **Sitemap** нужно раскомментировать и подставить домен.
- **`public/sitemap.xml`**: заглушка с одним URL `/about`; **example.com** заменить на боевой домен. Публичные страницы сервисов и шаринга лучше отдавать **отдельным динамическим sitemap** с Laravel (или генерировать при деплое).

**Важно:** это **SPA без SSR**. Поисковики всё чаще исполняют JS, но для максимальной стабильности индекса публичных URL позже можно добавить пререндер или SSR для `/about`, `/d/*`, `/g/*`, `/share/*`.

**Переменная окружения:** `VITE_SITE_ORIGIN` (см. `.env.example`) — опционально, если нужны абсолютные URL вне браузера.

## 4) Структура страниц — индекс / noindex

**Индексируемые (цель — поиск и переходы):**

| Маршрут | Комментарий |
|---------|-------------|
| `/about` | Маркетинговая «о сервисе». |
| `/d/:id` | Публичная страница детейлинга/СТО («на улице»). |
| `/g/:slug` | Публичный гараж, если **не** `garagePrivate`. |
| `/share/:token` | Публичная история одной машины. |

**Noindex (предложение, уже отражено в мета и robots):**

| Маршрут | Зачем |
|---------|--------|
| `/auth`, `/auth/*` | Вход и регистрация. |
| `/cars`, `/create` | Список авто и создание — личное. |
| `/garage`, `/garage/settings` | Кабинет владельца. |
| `/car/*` | Карточка, история, документы, редактирование. |
| `/detailing`, `/detailing/*` | Кабинет сервиса и настройки лендинга. |
| `/requests` | Заявки детейлинга. |

**Закрытый гараж** (`/g/...` при `garagePrivate`): в мета стоит **noindex** на странице-заглушке.

**Посадочные «детейлинг в Москве»:** отдельных URL в MVP нет. Варианты на будущее: статические страницы `/services/detyeling-moskva`, контент-хаб или фильтры на каталоге партнёров — после появления такого раздела.

## 5) Метаданные — зафиксированные правила

- **Title:** публичные сущности в начале, суффикс **`· КарПас`**; маркетинговая главная — **`КарПас — история обслуживания авто`** без дублирования суффикса.
- **Description:** до **~155 символов**, уникальный смысл по шаблону страницы, без обязательных юридических фраз.
- **Динамические шаблоны** — реализованы в коде для `/d/`, `/g/`, `/share/`.

---

## 6) Контент и сниппеты

- Тексты главной/FAQ уже есть на **`/about`**; тон — нейтрально-деловой, без обещаний «гарантии достоверности».
- **BreadcrumbList:** на публичной странице детейлинга в JSON-LD; при необходимости расширить на `/g/` и `/share/`.
- **FAQPage:** только на **`/about`** (синхронно с видимым FAQ).

Дальше: при росте трафика — расширить абзацы под запросы из п.3 (история авто, детейлинг + город), без переспама ключами.

---

## 7) Технические настройки на продакшене

- **Канонический домен:** один вариант (с www или без) + **301** на HTTPS; совпадает с `VITE_SITE_ORIGIN` / фактическим origin.
- **robots.txt** и **sitemap.xml** должны отдаваться с **боевого домена**; в `robots.txt` раскомментировать **Sitemap:**.
- Для **тысяч** публичных `/d/` лучше **sitemap-index** + несколько sitemap с API.

## 8) Аналитика и Search Console / Вебмастер

**Яндекс.Метрика (позже):** счётчик в `index.html` или через GTM; цели (JavaScript-события):

- успешный **вход** (владелец / партнёр);
- **регистрация** партнёра / отправка заявки;
- **добавление авто** (сохранение карточки);
- **запрос по VIN** / отправка заявки на привязку;
- **создание публичной ссылки** на авто;
- **клик «Позвонить»** на `/d/:id` (если не тянется из tel: автоматически).

**Google Search Console:** https://search.google.com/search-console — добавить ресурс (домен или префикс URL), подтвердить (DNS или файл).  
**Яндекс.Вебмастер:** https://webmaster.yandex.ru — добавить сайт, подтвердить, загрузить **sitemap**.

Сейчас ресурсов нет — после выкладки на домен добавить оба и отправить sitemap.

## 9) Локальное SEO

Карточки Яндекс/Google Business пока нет. Для детейлинга поля **NAP + часы** уже на публичной странице `/d/:id` — этого достаточно для старта; при появлении офлайн-точки — завести бизнес-профиль и совпадение данных с сайтом.

## 10) Политики, cookie-баннер

- Страницы политики/оферты — позже; **контакты** вы не выделяете отдельной страницей — ок, пока контакты на лендингах сервисов.
- **Cookie-баннер в РФ:** для собственных технических cookie сессии часто достаточно **политики** с перечислением cookie и целей; при подключении **Метрики/рекламы** — баннер с согласием на аналитику/маркетинг по модели, которую утвердит юрист (152-ФЗ + практика РКН). После включения Метрики — вынести перечень cookie и ссылку на политику в футер + баннер при первом визите.

## Семантика (п.3) — как использовать без переспама

Приоритетные темы: история автомобиля, детейлинг/СТО + **город**, обслуживание авто, VIN, бренд **КарПас** (варианты написания — в тексте и alt, не в одном title).  
Посадочные «услуга + город» (мойка Воронеж, детейлинг Москва) — через **отдельные страницы или раздел каталога**, когда появится контент; иначе риск тонких дублей.
